# Reel 39 - Canary Rollouts and Feature Flags

## Summary
Deploying new code to every user at once turns any defect into a full outage. The reel recommends canary traffic, feature flags, and automated promotion or rollback based on error rates.

## Verification Checklist
- Route a small percentage of traffic to new versions before full rollout.
- Put risky features behind flags that can be disabled without rollback.
- Use monitoring thresholds to auto-promote or auto-rollback canaries.

## Skill Signal
Use this as a safe-release scan for production deployment workflows.
