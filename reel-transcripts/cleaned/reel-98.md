# Reel 98 - Backups Only Count If Restores Work

## Summary
A backup file is not proof of recoverability. The reel recommends testing restores regularly, defining recovery point objectives based on actual data-loss tolerance, and measuring recovery time as a business-impact metric.

## Verification Checklist
- Run real restore drills on fresh instances instead of trusting that backups are usable.
- Define acceptable data-loss windows and ensure backup frequency meets them.
- Measure recovery time and confirm it aligns with the product’s actual downtime tolerance.

## Skill Signal
Use this as a backup-restore scan for RPO, RTO, and restore confidence.
