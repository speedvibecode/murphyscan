# Reel 151 - Build the Postmortem System Before the Incident

## Summary
Incident learning should be designed before production fails. A standard postmortem must capture impact, root cause, blast radius, remediation, and prevention; the review needs an owner and deadline; and completed reviews should form a searchable incident library that informs similar failures.

## Verification Checklist
- Maintain a postmortem template before the first incident.
- Record impact, root cause, blast radius, fix, and prevention actions.
- Assign an owner and time-bound review for every material incident.
- Track prevention actions through completion instead of stopping at documentation.
- Store reviews in a searchable incident library and consult it during related incidents.

## Skill Signal
Verify the full incident-learning loop: template, deadline, ownership, corrective actions, and reuse of prior evidence.
