# Reel 104 - Choose the Database by Workload, Not Brand

## Summary
Database choice should follow read-write profile, migration behavior under traffic, and portability goals. The reel recommends matching edge-oriented systems to read-heavy workloads, stronger write paths to concurrent mutation-heavy systems, and recognizing when an ecosystem choice is also a lock-in decision.

## Verification Checklist
- Compare database options against read-heavy versus write-heavy workload shape.
- Verify how schema changes are tested and applied under active production traffic.
- Distinguish between choosing a database engine and choosing a broader platform ecosystem commitment.

## Skill Signal
Use this as a database-workload scan for throughput fit and portability risk.
