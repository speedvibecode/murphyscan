# Reel 113 - AI Code Still Needs Engineering

## Summary
AI coding tools are growing quickly while developer trust remains low. The reel frames the gap between generated code and production trust as the reason for disciplined AI-directed engineering: the tool can write code, but the team still needs review, security scrutiny, testing, and production hardening.

## Verification Checklist
- Check whether AI-generated code receives the same or stronger review as human-written code in risky paths.
- Verify security-sensitive changes are reviewed for authorization, validation, secrets, dependency risk, and unsafe defaults.
- Confirm tests and runtime evidence cover generated behavior before it ships.
- Treat AI tool adoption as a need for stronger engineering discipline, not a replacement for it.
