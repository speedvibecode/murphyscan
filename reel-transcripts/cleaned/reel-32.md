# Reel 32 - Pick the Database by Data Shape

## Summary
There is no universal database choice. The reel recommends matching the database to data shape, ecosystem needs, and exit strategy: relational data fits Postgres, document-shaped data may fit Firebase or Convex, and proprietary systems increase migration cost.

## Verification Checklist
- Identify whether core data is relational, document-shaped, realtime, mobile-first, or edge-first.
- Evaluate surrounding ecosystem features, not only the storage engine.
- Check portability and exit cost before committing deeply.
