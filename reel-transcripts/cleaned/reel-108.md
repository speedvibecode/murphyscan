# Reel 108 - Watch the Failed Session

## Summary
Users often cannot describe bugs well enough for fast diagnosis. The reel recommends session replay, linking replays to error-tracking events, and detecting rage clicks so product failures are visible before support tickets become the only signal.

## Verification Checklist
- Verify session replay exists for the user-facing flows where support ambiguity creates meaningful risk.
- Confirm error events link to replay context so engineers can see what led to the exception.
- Check whether rage clicks, repeated failed actions, or stuck interactions are tracked as UX failures.
- Confirm replay collection is privacy-aware and does not expose sensitive user inputs unnecessarily.

Use this as an observability-depth scan for session replay, error context, and user-friction signals.
