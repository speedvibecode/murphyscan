# Reel 122 - Backups Are User Protection

## Summary
AI can ship a working product without ever raising data-loss questions. Real users create orders, messages, uploads, account settings, and payment history that cannot simply be rebuilt from source code. MurphyScan should treat backup schedule, retention, region/provider risk, and restore planning as user-protection controls.

## Verification Checklist
- Check whether user data has a backup schedule and retention policy.
- Verify backups are not stored only with the same failure domain as the primary database.
- Confirm the restore path is documented and tested before launch.
- Treat AI-built apps as especially likely to omit backup strategy unless asked explicitly.

Use this as a data backup strategy scan for user-data protection and AI-generated app blind spots.
