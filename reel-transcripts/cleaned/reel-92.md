# Reel 92 - Documentation Is Part of the System

## Summary
Teams forget architecture choices, local setup quirks, and failure responses unless those decisions are written down. The reel recommends documenting why the system looks the way it does, how to run it, and what breaks when a critical dependency fails.

## Verification Checklist
- Record major architecture decisions and the reasoning behind them.
- Document local setup steps, environment variables, and non-obvious workarounds.
- Maintain runbooks or playbooks for key failure modes instead of documenting only happy paths.

## Skill Signal
Use this as an operational-documentation scan for team continuity and incident readiness.
