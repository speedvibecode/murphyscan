# Reel 146 - Build Security Evidence Before Procurement

## Summary
Enterprise security review often begins before contract signature. Teams should be able to substantiate encryption, vulnerability scanning, penetration testing, incident response, data location, and vulnerability-reporting claims. A broad production audit should expose obvious control gaps before an external penetration test, with remediation and retesting producing evidence fit for procurement review.

## Verification Checklist
- Identify the security questionnaire and evidence expectations of target buyers.
- Verify encryption, scanning, incident-response, data-location, and vulnerability-reporting claims against implemented controls.
- Run a broad production-readiness audit before paying for a penetration test.
- Remediate audit findings, run an independent penetration test, and retest fixes.
- Keep a public security page or trust surface aligned with current evidence rather than aspirational claims.

## Skill Signal
Treat enterprise security procurement as an evidence-readiness check: control claims, audit results, remediation, and external testing must agree.
