# Reel 150 - Automate Known Support, Escalate Judgment Calls

## Summary
Support automation is safest for known issues with documented playbooks. Billing discrepancies, feature requests disguised as bugs, and emotionally charged repeat failures need human review because the correct response depends on external evidence, intent, history, empathy, or authority to make a business decision.

## Verification Checklist
- Define which support cases automation may resolve without review.
- Escalate when customer evidence conflicts with internal system state.
- Preserve account and incident history so a reviewer can see repeated failures.
- Route billing corrections, refunds, product-policy decisions, and high-risk sentiment to authorized humans.
- Measure false resolution, reopen, escalation, and churn-after-support rates.

## Skill Signal
Verify that support agents have explicit escalation boundaries and enough context for a human to recover ambiguous cases.
