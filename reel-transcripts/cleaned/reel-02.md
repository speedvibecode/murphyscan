# Reel 02 - First Security Baseline

## Summary
Most small AI-built apps ship without even the first layer of production security. The reel introduces a repeatable baseline: auth, encryption, sessions, error boundaries, rate limits, input validation, logging, backups, monitoring, and dependency scanning.

## Verification Checklist
- Confirm the app has a visible security checklist rather than ad hoc fixes.
- Check for coverage across auth, sessions, rate limiting, validation, logging, backups, and monitoring.
- Flag missing baseline controls as launch blockers, even when the app appears to work.

## Skill Signal
Use this as the top-level readiness scan that asks whether the repo has a repeatable production checklist.
