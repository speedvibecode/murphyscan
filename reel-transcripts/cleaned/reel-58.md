# Reel 58 - Branch Like You Deploy

## Summary
The main branch should represent production, not experimentation. The reel recommends starting with GitHub Flow: short-lived feature branches and pull requests, adding staging, release, or hotfix branches only when release complexity requires them.

## Verification Checklist
- Protect `main` as the production branch.
- Use short-lived feature or fix branches with pull requests before merge.
- Add GitFlow-style branches only when release cadence and risk justify the overhead.

Use this as a repository-process scan for branch discipline.
