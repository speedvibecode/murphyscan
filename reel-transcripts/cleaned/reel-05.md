# Reel 05 - Boring Deploys

## Summary
Deploying repeatedly until something works is gambling. The reel argues for preview deploys, small changes, and rehearsed rollback paths so every production release is deliberate and reversible.

## Verification Checklist
- Confirm every branch or PR gets a preview URL before production.
- Check that deploys are small enough to isolate failure causes.
- Verify rollback steps are known, tested, and fast enough for a real incident.

## Skill Signal
Use this as a deployment scan: inspect preview coverage, release size, and rollback readiness.
